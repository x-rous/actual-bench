# Actual Bench — Demo Backend

Runs `actualbudget/actual-server` (sync) + `jhonderson/actual-http-api` (REST) in
a single container. The demo budget is **baked into the image** and `start.sh`
re-copies it on every boot, so the demo always resets to a clean state.

This folder is the source for the Hugging Face Space
<https://huggingface.co/spaces/x-rous/actual-bench-demo>.

## Files

| File | Purpose |
|---|---|
| `Dockerfile` | Builds the combined sync + REST image (exposed on port 7860). |
| `start.sh` | Boots the sync server, waits, then starts the REST wrapper. |
| `seed-data/` | Pre-synced demo budget (generated by `generate-seed.mjs`). |
| `generate-seed.mjs` | Headless script that creates the seed budget + sample data. |

## Regenerating the seed budget

From the repo root:

```bash
node demo/generate-seed.mjs
```

It spins up a temporary local sync server, creates a "Demo Budget" with sample
accounts and transactions, writes the synced data to `demo/seed-data/`, and prints
the **Sync ID** to use as `DEMO_BUDGET_SYNC_ID` in Vercel. The server password it
uses is printed too — set it as the `ACTUAL_SERVER_PASSWORD` secret on the Space.

## Deploying to the Space

```bash
git clone https://huggingface.co/spaces/x-rous/actual-bench-demo
# copy README.md, Dockerfile, start.sh, seed-data/ into the clone
cd actual-bench-demo && git add . && git commit -m "demo backend" && git push
```

Then set Space secrets: `API_KEY` (any string → `DEMO_API_KEY` in Vercel) and
`ACTUAL_SERVER_PASSWORD` (the one printed by the seed script).

See `docs/DEMO_DEPLOYMENT.md` for the full end-to-end guide.
